# 🤝 راهنمای مشارکت

🌐 زبان: [English](CONTRIBUTING.md) | **فارسی**

از اینکه به مشارکت در این پروژه علاقه‌مند هستید، صمیمانه سپاسگزارم. ❤️

هدف این پروژه، ایجاد یک مسیر یادگیری ساختاریافته برای آموزش زبان برنامه‌نویسی پایتون از طریق تمرین‌ها، پروژه‌های عملی و سؤالات مصاحبه است.

چه بخواهید یک اشتباه تایپی را اصلاح کنید، مستندات را بهبود دهید یا تمرین جدیدی اضافه کنید، از مشارکت شما استقبال می‌شود.

---

## 📝 قبل از مشارکت

لطفاً پیش از ارسال تغییرات، موارد زیر را رعایت کنید:

- ساختار فعلی پروژه را حفظ کنید.
- کدهای پایتون را تمیز، خوانا و قابل فهم بنویسید.
- از استاندارد **PEP 8** پیروی کنید.
- در صورت امکان، برای کدها مثال ارائه دهید.
- از اضافه کردن تمرین‌ها یا پروژه‌های تکراری خودداری کنید.

---

## 📚 افزودن تمرین جدید

اگر قصد دارید تمرین جدیدی اضافه کنید:

- آن را در سطح مناسب قرار دهید:
  - 🟢 مبتدی
  - 🟡 متوسط
  - 🔴 پیشرفته

- مبحث مرتبط را به‌درستی انتخاب کنید.

- تا حد امکان از قالب استاندارد پروژه پیروی کنید.

هر تمرین بهتر است شامل بخش‌های زیر باشد:

- عنوان
- سطح دشواری
- شرح مسئله
- نمونه ورودی
- نمونه خروجی
- راهنمای حل (اختیاری)
- پاسخ
- توضیح راه‌حل

---

## 📂 ساختار پروژه

لطفاً ساختار پوشه‌های پروژه را بدون هماهنگی تغییر ندهید.

نمونه:

```text
01-Beginner/
└── 03-Loops/
    └── 01-For.md
```

## 🐞 گزارش مشکلات (Issues)

اگر با مشکلی مواجه شدید یا پیشنهادی برای بهبود پروژه دارید، لطفاً یک **Issue** ایجاد کنید و آن را با جزئیات توضیح دهید.

---

## 🔀 درخواست ادغام (Pull Request)

پیش از ارسال Pull Request لطفاً بررسی کنید:

- محتوای ارسالی تکراری نباشد.
- کدها بدون خطا اجرا شوند.
- تغییرات فقط روی یک موضوع متمرکز باشند.
- توضیح مناسبی برای Pull Request بنویسید.

---

## 💡 راه‌های مشارکت

شما می‌توانید از راه‌های زیر در توسعه پروژه مشارکت کنید:

- افزودن تمرین‌های جدید
- بهبود راه‌حل‌های موجود
- رفع اشکالات
- بهبود مستندات
- اصلاح اشتباهات نگارشی
- پیشنهاد توضیحات بهتر برای مفاهیم
